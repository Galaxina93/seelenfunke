php artisan serve --host 0.0.0.0 --port 8000

Backend Projekt Export:
php artisan project:export
php artisan project:export --short


App Projekt export:
dart run export_project.dart

App live aufs Handy releasen:
flutter run --release



DOCKER:

Container neu starten:
docker compose up -d

Alten Container stoppen:
docker compose down

Container neu bauen:
docker compose up -d --build

Container neu bauen ohne (Cache):
docker compose build --no-cache web

Container restlos löschen:
docker compose down -v

Artisan Migrate:
docker exec -it mein_php_server php artisan migrate
docker exec -it mein_php_server php artisan migrate:fresh --seed

Schreibrechte unter Linux:
chmod -R 777 storage bootstrap/cache

Mach den aktuellen Benutzer zu dem Besitzer von allen Dateien und Ordnern:
sudo chown -R $USER:$USER .

CHECKOUT:
Worker starten um Jobs zu testen:
Da wir im Docker Container arbeiten müssen wir den Befehl auch im Container abfeuern:
1. Berechtigung setzen: chmod +x vendor/bin/sail
2. Worker starten: ./vendor/bin/sail artisan queue:work


Chat Event Starten:
funki artisan reverb:start

Wenn der Chat mal wieder nicht geht denk dran:
Das ist der häufigste Fehler bei Reverb und Vite: Die Variablen aus deiner
.env (wie VITE_REVERB_HOST=127.0.0.1) werden beim Bauen fest in deine app.js
Datei eingebrannt.
Wenn du Dateien zurückgesetzt hast, ist deine app.js vielleicht noch auf
dem Stand von unseren Stage-Tests und versucht gerade lokal,
sich mit wss://ws.mein-seelenfunke.de zu verbinden!


Mittwald WORKER neustarten:
php artisan queue:restart

Bei Env File Änderungen danach immer:
php artisan optimize:clear
